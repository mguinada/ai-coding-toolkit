## Setup

### Directory Structure
```
config/locales/
├── en.yml
├── es.yml
└── views/
    └── users/
        ├── en.yml
        └── es.yml
```

### Default Locale
```ruby
# config/application.rb
config.i18n.default_locale = :en
config.i18n.available_locales = [:en, :es, :fr, :de]
config.i18n.load_path += Dir[Rails.root.join("config", "locales", "**", "*.{rb,yml}")]
```

## Looking Up Translations

### In Views
```erb
<%= t("hello") %>
<%= t("site.name") %>

<!-- With scope -->
<%= t(".title") %>  <!-- Looks up users.index.title -->
<%= t("users.index.title") %>
```

### In Controllers/Models
```ruby
I18n.t("hello")
I18n.t("site.name")

# With default
I18n.t("missing_key", default: "Default text")
I18n.t("missing_key", default: "some.existing.key")
```

### Bulk Lookup
```ruby
# Returns hash of translations
I18n.t("site")
# => { name: "My App", tagline: "Build something amazing" }
```

## Interpolation

### Basic Interpolation
```yaml
en:
  greeting: "Hello, %{name}!"
  messages:
    welcome: "Welcome to %{site}, %{user}!"
```

```erb
<%= t("greeting", name: "John") %>
<%= t("messages.welcome", site: "MyApp", user: @user.name) %>
```

### With Objects
```ruby
I18n.t("greeting", name: user.name)
```

## Pluralization

### Basic Pluralization
```yaml
en:
  inbox:
    one: "1 message"
    other: "%{count} messages"
```

```erb
<%= t("inbox", count: 1) %>   <!-- 1 message -->
<%= t("inbox", count: 5) %>   <!-- 5 messages -->
```

### Zero and Other Rules
```yaml
en:
  messages:
    zero: "No messages"
    one: "1 message"
    other: "%{count} messages"
```

### With Interpolation
```yaml
en:
  unread:
    one: "%{count} unread message from %{sender}"
    other: "%{count} unread messages from %{sender}"
```

```erb
<%= t("unread", count: 3, sender: "John") %>
```

### Russian Pluralization (Complex)
```ruby
# config/initializers/pluralization.rb
I18n.backend.store_translations(:ru, {
  i18n: {
    plural: {
      keys: %i[one few many other],
      rule: ->(n) {
        n % 10 == 1 && n % 100 != 11 ? :one :
        [2, 3, 4].include?(n % 10) && ![12, 13, 14].include?(n % 100) ? :few :
        n % 10 == 0 || [5, 6, 7, 8, 9].include?(n % 10) || [11, 12, 13, 14].include?(n % 100) ? :many : :other
      }
    }
  }
})
```

## Locale Management

### Setting Locale
```ruby
# Around action
around_action :switch_locale

def switch_locale(&action)
  locale = params[:locale] || session[:locale] || I18n.default_locale
  I18n.with_locale(locale, &action)
end

# Or in before_action
before_action :set_locale

def set_locale
  I18n.locale = extract_locale || I18n.default_locale
end

def extract_locale
  params[:locale] ||
    session[:locale] ||
    request.env["HTTP_ACCEPT_LANGUAGE"]&.scan(/^[a-z]{2}/)&.first
end
```

### Locale from URL
```ruby
# config/routes.rb
scope "(:locale)", locale: /en|es|fr/ do
  resources :articles
  root "home#index"
end
```

```ruby
# app/controllers/application_controller.rb
def default_url_options
  { locale: I18n.locale }
end
```

### Locale from Domain
```ruby
def extract_locale_from_tld
  request.domain.split(".").last
end

# es.example.com -> :es
# fr.example.com -> :fr
```

### Locale from Subdomain
```ruby
def extract_locale_from_subdomain
  request.subdomains.first
end

# es.example.com -> :es
```

## Date/Time Localization

### Date Formats
```yaml
en:
  date:
    formats:
      default: "%Y-%m-%d"
      short: "%b %d"
      long: "%B %d, %Y"
```

```erb
<%= l(Date.today) %>                    <!-- 2024-01-15 -->
<%= l(Date.today, format: :short) %>    <!-- Jan 15 -->
<%= l(Date.today, format: :long) %>     <!-- January 15, 2024 -->
```

### Time Formats
```yaml
en:
  time:
    formats:
      default: "%a, %d %b %Y %H:%M:%S %z"
      short: "%d %b %H:%M"
      long: "%B %d, %Y %H:%M"
```

```erb
<%= l(Time.current) %>                   <!-- Mon, 15 Jan 2024 14:30:00 -0500 -->
<%= l(Time.current, format: :short) %>  <!-- 15 Jan 14:30 -->
```

### Day and Month Names
```yaml
en:
  date:
    day_names: [Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday]
    abbr_day_names: [Sun, Mon, Tue, Wed, Thu, Fri, Sat]
    month_names: [~, January, February, March, ...]
    abbr_month_names: [~, Jan, Feb, Mar, ...]
```

### Custom Formats in Code
```ruby
I18n.l(date, format: "%B %Y")  # January 2024
I18n.l(time, format: :db)      # 2024-01-15 14:30:00
```

## Active Record Localization

### Model Names
```yaml
en:
  activerecord:
    models:
      user: "User"
      article: "Article"
```

```ruby
User.model_name.human          # => "User"
User.model_name.human(count: 2) # => "Users"
```

### Attribute Names
```yaml
en:
  activerecord:
    attributes:
      user:
        email: "Email address"
        password: "Password"
        created_at: "Signed up"
```

```ruby
User.human_attribute_name(:email)      # => "Email address"
User.human_attribute_name("address.city")  # Nested attributes work too
```

### Error Messages
```yaml
en:
  activerecord:
    errors:
      models:
        user:
          attributes:
            email:
              blank: "Please enter your email"
              invalid: "Please enter a valid email"
          taken: "This email is already registered"
```

### Default Error Messages
```yaml
en:
  errors:
    format: "%{attribute} %{message}"
    messages:
      blank: "can't be blank"
      present: "must be blank"
      confirmation: "doesn't match %{attribute}"
      accepted: "must be accepted"
      empty: "can't be empty"
      equal_to: "must be equal to %{count}"
      even: "must be even"
      odd: "must be odd"
      greater_than: "must be greater than %{count}"
      greater_than_or_equal_to: "must be greater than or equal to %{count}"
      inclusion: "is not included in the list"
      invalid: "is invalid"
      less_than: "must be less than %{count}"
      less_than_or_equal_to: "must be less than or equal to %{count}"
      not_a_number: "is not a number"
      not_an_integer: "must be an integer"
      taken: "has already been taken"
      too_long: "is too long (maximum is %{count} characters)"
      too_short: "is too short (minimum is %{count} characters)"
      wrong_length: "is the wrong length (should be %{count} characters)"
```

## Action Mailer Localization

### Subject Translations
```yaml
en:
  user_mailer:
    welcome_email:
      subject: "Welcome to MyApp!"
    password_reset:
      subject: "Reset your password"
```

```ruby
class UserMailer < ApplicationMailer
  def welcome_email
    # Subject automatically translated from user_mailer.welcome_email.subject
    mail(to: @user.email)
  end
end
```

### Email Body Localization
```ruby
class UserMailer < ApplicationMailer
  def welcome_email(user)
    @user = user
    I18n.with_locale(@user.locale) do
      mail(to: @user.email)
    end
  end
end
```

## Number Localization

### Currency
```yaml
en:
  number:
    currency:
      format:
        format: "%u%n"
        unit: "$"
        separator: "."
        delimiter: ","
        precision: 2
```

```erb
<%= number_to_currency(1234.56) %>         <!-- $1,234.56 -->
<%= number_to_currency(1234.56, unit: "€") %> <!-- €1,234.56 -->
```

### Percentage
```erb
<%= number_to_percentage(66.67) %>  <!-- 66.67% -->
```

### Delimited
```erb
<%= number_with_delimiter(1234567) %>  <!-- 1,234,567 -->
```

### Human Size
```erb
<%= number_to_human_size(1234567) %>  <!-- 1.18 MB -->
```

## Best Practices

1. **Use lazy lookups** - `t(".title")` in views for cleaner code
2. **Organize by feature** - Separate locale files for different features
3. **Avoid hardcoded strings** - Always use `t()` helper
4. **Use pluralization** - Don't create separate keys for singular/plural
5. **Set locale early** - Use `around_action` for consistent locale
6. **Persist locale preference** - Store in session or database
7. **Test all locales** - Ensure translations exist for all keys
8. **Use i18n-tasks gem** - Find missing/unused translations
